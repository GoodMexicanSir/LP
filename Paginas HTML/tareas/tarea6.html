Haskell

Símbolos, sintáxis, esqueleto de los programas

Símbolos:

Clases de tipos:
  • El tipo Bool
    • (&&):: Bool -> Bool -> Bool.
    • (||):: Bool -> Bool -> Bool.
    • not:: Bool -> Bool -> Bool.
    • otherwise:: Bool.
  • El tipo Int
    • (+),(-),(*):: Int -> Int -> Int.
    • (↑):: Int -> Int -> Int.
    • div, mod:: Int -> Int -> Int.
    • abs:: Int -> Int.
    • signum:: Int -> Int.
    • negate:: Int -> Int.
    • even, odd:: Int -> Bool.
  • El tipo Float
    • (+),(*),(-),(/)
    • :: Float -> Float -> Float.
    • (↑):: Float -> Float -> Float.
    • (**):: Float -> Float -> Float.
    • sin, asin, cos, acos, tan, atan:: Float -> Float -> Float.
    • atan2:: Float -> Float -> Float.
    • log, log10, exp:: Float -> Float -> Float.
    • sqrt:: Float -> Float -> Float.
  • El tipo Char
    • ord:: Char -> Int.
    • chr:: Int -> Char.
    • isUpper, isLower, isDigit, isAlpha:: Char -> Bool.
    • toUpper, toLower:: Char -> Char.
  • Eq es utilizada por los tipos que soportan comparaciones por igualdad.
  • Ord es para tipos que poseen algún orden.
  • Show pueden ser representados por cadenas.
  • Read es como la clase de tipos opuesta a Show.
  • Enum son tipos secuencialmente ordenados, es decir, pueden ser enumerados.
  • Bounded poseen límites inferiores y superiores, es decir están acotados.
  • Num es la clase de tipos numéricos. Sus miembros tienen la propiedad de poder comportarse como números.
  • Integral es también un clase de tipos numérica. Num incluye todos los números, incluyendo números reales y enteros.
  • Floating incluye únicamente números en coma flotante, es decir Float y Double.
Sintáxis:
  • Comentarios de una sola l ́ınea: comienzan por dos guiones consecutivos (– –) y abarcan hasta el final de la l ́ınea actual
  • Comentarios que abarcan m as de una línea: Comienzan por los caracteres {− y acaba con −}. Pueden abarcar varias líneas y anidarse
  • Este lenguaje identifica el tipo de dato por si mismo, para verificarlo se puede escribir :t
    • 	ghci> :t 'a'
	'a' :: Char
	ghci> :t True
	True :: Bool
	ghci> :t "HOLA!"
	"HELLO!" :: [Char]
	ghci> :t (True, 'a')
	(True, 'a') :: (Bool, Char)
	ghci> :t 4 == 5
	4 == 5 :: Bool
  • Las funciones también tiene tipos. Cuando escribimos nuestras propias funciones podemos darles un tipo explícito en su declaración.
    • 	removeNonUppercase :: [Char] -> [Char]
	removeNonUppercase st = [ c | c <- st, c `elem` ['A'..'Z']]
Esqueleto:

Una vez instalado el compilador se pueden escribir directamente las instrucciones, por ejemplo:
	GHCi, version 8.2.1: http://www.haskell.org/ghc/  :? for help
	Prelude> 2 + 15
	17
Donde 2 + 15 es la instrucción y 17 es el resultado de la ejecución.

Ejemplo de programa

Función simple que realiza el cálculo bajo las instrucciones dadas, con una pequeña modificación a la función: 
	GHCi, version 8.2.1: http://www.haskell.org/ghc/  :? for help
	Prelude> doubleSmallNumber x = if x > 100
        		then x
                else x*2
	Prelude> doubleSmallNumber' x = (if x > 100 then x else x*2) + 1
	Prelude> doubleSmallNumber' 30
	61
	Prelude>
